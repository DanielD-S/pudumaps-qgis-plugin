changelog=0.7.4 (2026-05-18) — Landcover + re-etiquetado clases Chile
  * Tercera acción IA: "Clasificar uso de suelo"
    (LandCoverClassificationTool). Produce raster GeoTIFF categórico
    con sidecar JSON de leyenda re-etiquetada al español + ecorregión
    chilena cuando el bbox cae en Chile.
  * Nuevo módulo pudumaps_qgis.ai.chile_classes (Nivel 1 chilenización
    en código): lookup determinístico bbox → ecorregión (matorral
    esclerófilo / estepa patagónica / bosque siempreverde / desierto
    costero / etc.) + traducción de clases genéricas (shrubland →
    matorral esclerófilo / patagónico según zona).
  * AITool gana atributo output_suffix (default .geojson). Landcover
    usa .tif. El panel ahora dispatcha QgsVectorLayer / QgsRasterLayer
    por extensión del output.

 0.7.3 (2026-05-18) — Ejecución async + extract_water + doc Chile
  * Las acciones IA ahora corren en QgsTask (background thread del task
    manager de QGIS). La UI ya no se congela durante la inferencia y
    aparece una entry en el panel Tasks con progreso y cancelación.
  * Nueva acción: "Detectar cuerpos de agua" (ExtractWaterTool) — para
    lagos, embalses y ríos sobre raster RGB. Funciona con Sentinel-2 y
    ortofoto. Caveats Chile documentados.
  * Nuevo docs/modelos-chile.md (Nivel 1 chilenización): tabla de qué
    casos rinden bien con pesos de fábrica y cuáles requieren
    finetuning (rural, palafitos, tomas, bosque nativo).

 0.7.2 (2026-05-18) — Panel IA + primera acción (extract_buildings)
  * Nuevo framework pudumaps_qgis/ai/tools con clase abstracta AITool,
    registry estático y manejo de dependencias por tool.
  * Primera acción: "Detectar edificaciones" sobre raster RGB usando
    geoai.BuildingFootprintExtractor. Aislada la llamada a geoai en un
    helper único para que el upgrade entre versiones del paquete sea
    localizado.
  * Panel lateral "Pudumaps · IA" como QDockWidget (toggle desde menú
    "Panel IA"). Lista las acciones registradas, grisa botones si
    falta geoai y muestra mensaje invitando al instalador.
  * Resultado de cada acción se carga automáticamente como capa
    vectorial en el proyecto QGIS.

 0.7.1 (2026-05-18) — Instalador IA en menú
  * Nueva entrada "Instalar módulo IA…" en el menú Pudumaps y la
    toolbar, con icono propio (ai.svg). Abre el diálogo de instalación
    introducido en 0.7.0 para que sea descubrible por el usuario sin
    tener que invocarlo desde código.

 0.7.0 (2026-05-18) — Infra IA opcional (foundation)
  * Nuevo módulo pudumaps_qgis.ai con detección de geoai-py y GeoAgent
    como dependencias opcionales. Versiones pineadas (geoai-py 0.10.0,
    GeoAgent 0.4.0) para protegerse de bumps rotos upstream.
  * Instalador asistido (pip install --user) contra el Python embebido
    de QGIS, con progreso línea a línea en un worker thread.
  * Diálogo "Instalar módulo IA" que explica tamaños (~500 MB descarga,
    ~2 GB en disco) y prerequisitos (VC++ redist en Windows).
  * Sin acciones IA todavía — esta release solo prepara la
    infraestructura. Las acciones (detección de edificios, agua,
    landcover, change detection, descarga Sentinel) llegan en 0.7.1.

 0.6.0 (2026-05-07) — Security hardening (audit 2026-05-07)
  * H1 ALTO: prompt explícito antes de degradar a almacenamiento
    plaintext de QSettings cuando QgsAuthManager no tiene master
    password. Antes el fallback era silencioso.
  * H2 ALTO: rechazo de URLs base no-HTTPS (excepto loopback). Previene
    leak de la API key si el user/atacante setea http://evil.example/
  * H3 MEDIO: tempfile usado por OGR ahora se borra en finally. Solo
    sobrevive cuando OGR es la layer activa y necesita leerlo on-demand
    (caso fallback).
  * H4 MEDIO: cap de 50 MB en respuestas HTTP + validación estructural
    de GeoJSON antes de pasar a OGR. Previene OOM por payloads
    malformados.
  * H5 MEDIO: helper safe_error_message en todos los diálogos. Stack
    traces y mensajes técnicos de la API ya no se muestran al usuario;
    el detalle queda en QgsMessageLog para debugging.
  * H6 BAJO: User-Agent leído dinámicamente desde metadata.txt. Antes
    estaba hardcoded a 0.1.0.
  * H8 BAJO: clear_credentials retorna ahora True/False según pudo
    eliminar la entry cifrada. SettingsDialog avisa al user si quedó
    residuo cifrado por falta de master password en sesión.

 0.5.0 (2026-04-23)
  * Logo oficial Pudumaps (el pudu con mapa) integrado como icono del

  * Iconos SVG específicos por acción: settings (engranaje), download

 0.4.0 (2026-04-23)
  * Sincronización bidireccional con detección de conflictos
  * Diálogo de sync con tabla por capa + dropdown de acción
  * Estados: unchanged / local_only / remote_only / conflict / new_local /
    deleted_remote
  * Hashes SHA-256 canónicos persistidos como pudumaps/last_hash
  * Dedup en pull: si ya existe una capa con el mismo layer_id, se
    actualizan sus features en vez de crear un duplicado

 0.3.0 (2026-04-23)
  * Subir capas QGIS a Pudumaps (push)
  * Menú contextual: click derecho en capa vectorial → Subir a Pudumaps
  * Acción del menú: Subir capa activa a Pudumaps
  * Diálogo con dropdown de proyectos + creación inline de proyecto nuevo
  * Reproyección automática a EPSG:4326 desde cualquier CRS soportado
  * Pre-validación de tamaño (10 MB) y número de features (20k)
  * Smart dispatch: capas que vinieron del pull se actualizan in-place
    (PATCH) en vez de duplicarse (POST)

 0.2.1 (2026-04-23)
  * Fix: polygon/multipolygon layers no renderizaban — OGR promueve a
    Multi* y el memory layer Polygon rechazaba las features. Ahora se
    usa el wkbType real de OGR.
  * Fix: CRS del memory layer ahora se hereda del OGR layer en vez
    de hardcodear EPSG:4326.
  * Auto-zoom al grupo recién cargado para que las capas sean visibles
    aunque estén lejos del viewport inicial.

 0.2.0 (2026-04-23)

  * Automatic layer group "Pudumaps: <project>" in layer tree
  * Basic default styling (green) applied to each loaded layer
  * Remote project/layer ids stamped as custom properties for future sync
