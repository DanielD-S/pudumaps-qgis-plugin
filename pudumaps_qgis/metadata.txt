changelog=0.3.0 (2026-04-23)
  * Subir capas QGIS a Pudumaps (push)
  * Menú contextual: click derecho en capa vectorial → Subir a Pudumaps
  * Acción del menú: Subir capa activa a Pudumaps
  * Diálogo con dropdown de proyectos + creación inline de proyecto nuevo
  * Reproyección automática a EPSG:4326 desde cualquier CRS soportado
  * Pre-validación de tamaño (10 MB) y número de features (20k)
  * Smart dispatch: capas que vinieron del pull se actualizan in-place
    (PATCH) en vez de duplicarse (POST)

 0.2.1 (2026-04-23)
  * Fix: polygon/multipolygon layers no renderizaban — OGR promueve a
    Multi* y el memory layer Polygon rechazaba las features. Ahora se
    usa el wkbType real de OGR.
  * Fix: CRS del memory layer ahora se hereda del OGR layer en vez
    de hardcodear EPSG:4326.
  * Auto-zoom al grupo recién cargado para que las capas sean visibles
    aunque estén lejos del viewport inicial.

 0.2.0 (2026-04-23)
  * Open a Pudumaps project as QGIS layers (pull)
  * Projects dialog with list, search-by-sort, double-click to open
  * Automatic layer group "Pudumaps: <project>" in layer tree
  * Basic default styling (green) applied to each loaded layer
  * Remote project/layer ids stamped as custom properties for future sync

 0.1.0 (2026-04-23)
  * Initial release
  * Settings dialog with encrypted API key storage via QgsAuthManager
  * Connection test against /v1/projects
